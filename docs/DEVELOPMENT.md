# 避障版开发规划

> 主程序：`car_tb6612_avoid/car_tb6612_avoid.ino`  
> 硬件基线见根目录 [README.md](../README.md)。

## 目标概览

在现有「单探头朝前 + 时间转向 + 自适应脱困」基础上，分阶段升级为：**舵机扫描 → 状态机 → 转向记忆 → 右手法则 → 编码器定角**。

| 阶段 | 内容 | 状态 |
|------|------|------|
| **1** | 舵机扫描：HC-SR04 装在舵机上，遇障扇区测距 | **进行中 / 本阶段交付** |
| **2** | 状态机：`FORWARD` / `BACKWARD` / `TURN_LEFT` / `TURN_RIGHT` / `SCAN` | 未开始 |
| **3** | 转向避障记录：记录每次脱困方向与结果，供下次决策 | 未开始 |
| **4** | 右手法则：优先右转 → 否则直行 → 否则左转 → 否则后退 | 未开始 |
| **5** | 编码器：用轮速脉冲控制转向角度，替代纯 `delay` 转角 | 未开始 |

## 阶段 1：舵机扫描（当前）

### 硬件

| 部件 | 接法 |
|------|------|
| SG90 类舵机信号线 | **D11**（5V / GND 与模块共地） |
| HC-SR04 | 随舵机云台一起转；**Trig=D9，Echo=D12** 不变 |

安装建议：

- 舵机中位（约 90°）时超声波两圆孔**水平朝前**，与车头平行。
- 扫描范围建议左右各约 30°～40°（程序默认 5 个角度：约 55°～125°，中位 90°）。
- 舵机供电尽量从电源模块 5V 取，**信号线仍接 D11**；大电流舵机与 Uno 共 5V 时若复位，给舵机单独供电并共地。

### 软件行为（阶段 1）

- 宏 `USE_SERVO_SCAN=1`（默认）：直行时舵机保持 **90°** 朝前测距；遇障停稳后 **扇区扫描**，按左右最远距离选转向（**同距优先右转**，为阶段 4 预留）。
- 宏 `USE_SERVO_SCAN=0`：与旧版相同，遇障仍 **左右交替** 转向（`lastTurnDir` 取反）。
- 调试：`DEBUG_SCAN_ONLY=1` 只扫描、电机不动，串口打印各角度 `cm`。
- **未改**：自适应脱困档位（`stuckLevel`）、后退/转向仍用 **时间**（阶段 5 再改编码器）。

### 验收

1. `DEBUG_SCAN_ONLY=1`：串口可见 5 个角度距离，手挡左/右侧数值应明显变化。
2. `DEBUG_SCAN_ONLY=0`：遇障应倾向转向**较空**一侧；窄道仍可能出不来（单探头局限不变）。

## 阶段 2：状态机（规划）

将 `loop()` 中「前进 / 后退 / 转向 / 扫描」拆为显式状态，便于串口观测与后续接右手法则、编码器。

```
enum RobotState {
  FORWARD,
  BACKWARD,
  TURN_LEFT,
  TURN_RIGHT,
  SCAN
};
```

| 状态 | 预期行为 |
|------|----------|
| `FORWARD` | 前方畅通则低速直行 |
| `SCAN` | 停车、舵机扫一圈（或扇区），写入距离表 |
| `BACKWARD` | 遇障或死胡同后退一段 |
| `TURN_LEFT` / `TURN_RIGHT` | 按决策转向（阶段 5 改为编码器定角） |

迁移思路：阶段 1 的 `scanEnvironment()` 归入 `SCAN`；现有 `runAdaptiveAvoidance()` 拆成 `BACKWARD` + `TURN_*` 转移。

---

## 阶段 3：转向避障记录（规划）

维护简短历史，避免「左一下右一下回到原位」或重复撞同一侧。

建议记录字段（RAM 紧张，用位域/定长环形缓冲）：

| 字段 | 说明 |
|------|------|
| `turnDir` | 上次左转 / 右转 |
| `stuckLevel` | 已有，可并入记录 |
| `outcome` | 转向后前方是否畅通、多久再次遇障 |
| `scanSnapshot` | 可选：左/中/右三档最大距离（压缩存储） |

用途：阶段 4 决策时，若右手法则与历史冲突（例如连续右转仍堵），升档或强制反向。

---

## 阶段 4：右手法则（规划）

在 `SCAN` 结果上按固定优先级选动作（迷宫贴墙常用策略，**不保证全局最优**）：

1. **优先右转** — 右侧扇区距离 ≥ 阈值  
2. **否则直行** — 正前方畅通  
3. **否则左转** — 左侧更空  
4. **否则后退** — 三面皆堵，后退并升 `stuckLevel`

与阶段 1 关系：阶段 1 遇障扫描后仅「左右选较空 + 同距偏右」；阶段 4 在状态机里完整实现四选一，且直行也参与决策。

## 阶段 5：编码器定角（规划）

### 硬件（待采购/接线）

| 项目 | 说明 |
|------|------|
| 编码器 | 左右驱动轮各 1 路，或仅从动轮；需与现有 TB6612 接线不冲突 |
| 引脚 | 待选型（避开 D3～D12 已占用；可考虑 A0～A5 外部中断） |

### 软件

- 标定：`PULSES_PER_90DEG`（或每度脉冲数）在平地架空测定。
- `TURN_LEFT` / `TURN_RIGHT`：目标脉冲数 = `f(angle)`，替代 `MS_TURN_*` 固定延时。
- 与 `stuckLevel`：档位映射为 **目标角度** 而非 `delay` 加长。
