## 方向调整说明

最初规划是**舵机扫描 + 扇区测距**，但实际买到的是 **360° 连续旋转舵机**（无绝对角度反馈），方案改为：**固定朝前 HC-SR04 + 状态机 + 右手法则**。详见阶段路线。

## 已知局限 — 当前硬件下暂不解决（记入后续升级）

> 平台 **`uno_tb6612_hc04`**：仅 1 路 HC-SR04 **固定朝前**，无侧面/后方测距、无碰撞条、无编码器里程。  
> 下列问题**单靠改 ALG-1.x 参数或状态机无法根治**；文档记录现象与目标，待硬件升级后再做（如 ALG-2.0 / 侧向传感器）。

### 窄缝卡死 — 前空身卡、持续顶进（实车观察）

| 项 | 说明 |
|----|------|
| **现象** | 多障碍物形成的**窄缝**中，探头前方测距**畅通**（`FORWARD` / `cm ≥ dSafe`），但**车宽或对角已被两侧挡住**；小车仍低速/中速**向前顶**，无法自行脱困 |
| **原因** | 决策只认「车头波束方向距离」，不认**整车包络**是否卡在缝里；无侧向距离 → 不知道左右翼子板已顶墙 |
| **与单障区别** | 单柱可右/左探绕开；窄缝常需**横向脱出或后退+大角**，前向超声给不出「侧身被夹」的信息 |
| **当前策略** | **不纳入 ALG-1.3 必做项**；窄道场景避免实测或人工搬出 |
| **后续硬件/算法方向** | ① 可转扫描（180° 舵机 + HC-SR04，ALG-2.0）看两侧是否同时很近；② 左右红外/超声测距；③ 碰撞条触发强制 BACKUP+ESCAPE；④ 编码器/陀螺：前进但距离不变 → 判定卡死 |

### 侧面刮蹭 — 前通侧不通（与单障 11 次相关）

探头方向 `cm ≥ dProbeOk` 时，**车体左/右侧**仍可能碰障（见上文「实车记录 ×11」）。缓解靠 1.3 的 `dProbePass`、斜走回正；**彻底避免**仍需侧向感知或更大横向余量。

## 直线跑道 + 单障碍物 — 实车问题与共识（2026-05）

### 现象（ALG-1.1）

| # | 现象 | 主要原因（当前实现） |
|---|------|----------------------|
| 1 | 转向幅度过大 | 一次右/左探 **35°**；ESCAPE **90°+**；通过后**保持探角**不复正 |
| 2 | 后退过长 | `MS_BACKUP_BASE` 350ms，升档可达 900ms |
| 3 | 脱困后再次进入 | 朝向偏了仍用 `dProbeOk≈44cm` 判通；障碍仍在波束内 |
| 4 | 难保持跑道方向 | 设计为「选通方向并保持」，非「绕障后回正」 |
| 5 | 判通距离偏高 | `dProbeOk = dSafe+5`（约 44cm）要求正前方很空，**20cm 柱心**时转小角后仍易判不通 |

### 测试约定

- **障碍**：正对柱心测距。
- **路径**：允许**斜走一段再回正**（跑道模式，与迷宫贴墙模式区分）。

### 改进方向（分步）

| 优先级 | 内容 | ALG 版本 |
|--------|------|----------|
| P0 | 缩短 `BACKUP` / `ESCAPE` 后退时长 | **1.2 ✅** |
| P0 | 右探 **15→30→…→90°** 阶梯，用尽回正再左探同样阶梯 | **1.2 ✅** |
| P1 | **`dProbePass(累计转角)`** 探测判通 | **1.3 ✅** |
| P1 | 探通后 **WEAVE 短前进 + RECENTER 回正** | **1.3 ✅** |
| P2 | **过障完成**（前方连续变远 / 回正后 `dClear` 持续）再清零 `stuckLevel` | 1.4 规划 |
| P2 | 转向避障记录，避免重复 ESCAPE | 1.4 规划 |

> 阶梯探测每档需停转测距，Uno 上较慢但转角更小；不必在 DECIDE 内扫满 90°×2 才 BACKUP，右/左各 3～4 档 + ESCAPE 通常够用。

## ALG-1.2（`fsm_rhr_step`）— 已实现

### Sketch / 参数

| 项 | 路径 |
|----|------|
| 固件 | `hardware/uno_tb6612_hc04/uno_tb6612_hc04_avoid_v2_fsm_alg_1_2/` |
| 参数 | `params_alg_1_2_cfg_1_0.h`（基线）/ **`params_alg_1_2_cfg_1_1.h`（跑道，当前默认）** |

### 相对 ALG-1.1 的代码差异

**后退（cfg_1_0）**

| 参数 | 1.1 | 1.2 |
|------|-----|-----|
| `MS_BACKUP_BASE` | 350 ms | **200 ms** |
| `MS_BACKUP_STEP` | 150 ms | **100 ms** |
| `MS_BACKUP_MAX` | 900 ms | **550 ms** |

**DECIDE 探测**

- 1.1：右 **35°** → 回正 → 左 **35°** → ESCAPE  
- 1.2：右 **15°×6 档**（累计 90°），每档测距；右用尽 **回正** → 测前方 → 左 **15°×6** → ESCAPE  
- 判通阈值仍为 **`dProbeOk`**（与 1.1 相同，待 1.3 拆 `dProbePass`）

**串口**

- 上电打印 `ALG-1.2`、`probe steps`、`BACK ms`  
- `DEC ph=` / `probe R step=` / `decide=R step OK` 等

### DECIDE 流程（1.2）

```
ENTRY → 右 +15° → 测 → 不通则 +15° … 至 90°
      → 右用尽：回正 → CENTER 测
      → 仍不通：左 +15° … 阶梯
      → 左用尽：UNWIND → ESCAPE
```

### cfg-1.1（跑道 / 新电池，2026-05）

| 项 | cfg-1.0 | cfg-1.1 |
|----|---------|---------|
| 单侧最大探角 | 90°（6×15°） | **45°（3×15°）** |
| `DEG_PER_MS` | common **0.105** | 覆盖 **0.118**（新电池转角偏大时缩短 ms） |
| `ESCAPE_TURN_DEG` | 90 | **60** |
| `MS_BACKUP_BASE` | 200 | **180** |

sketch 默认 `#include params_alg_1_2_cfg_1_1.h`；对比凌晨数据时改回 `cfg_1_0` 即可。

**DECIDE 顺序（1.2 修复）**：右探用尽 → 回正 → **正前测距 → 左探阶梯**；回正后若 `cm<dStop` **不再**误进 BACKUP（极近 BACKUP 仅在 `PH_DECIDE_ENTRY`）。

### 实车记录（ALG-1.2 cfg-1.1，直线单障 ×11）

| 结果 | 说明 |
|------|------|
| 11/11 通过 | 阶梯 45° + 短后退 + 回正后左探顺序有效 |
| 约 4/11 | 先右探若干档不通 → 左探 1～2 档后通过；**车身右侧蹭柱** |

**原因（非随机）**：仅 1 路 HC-SR04 朝**探头波束方向**测距，不覆盖**车宽两侧包络**。右探失败后左探通过时，判据是「当前朝向的 cm ≥ dProbeOk」，**未校验车体右缘与柱的横向间隙**（`CAR_WIDTH` 未参与通过判定），故易出现「探头方向够宽、右侧仍刮蹭」。

**后续**：ALG-1.3 已含 `dProbePass` + WEAVE/RECENTER；侧向刮蹭仍可能，见「已知局限」。

## ALG-1.3（`fsm_pass_weave`）— 已实现

| 项 | 路径 |
|----|------|
| Sketch | `hardware/uno_tb6612_hc04/uno_tb6612_hc04_avoid_v2_fsm_alg_1_3/` |
| 参数 | `params_alg_1_3_cfg_1_0.h` / **`params_alg_1_3_cfg_1_1.h`（默认）** |

### 相对 1.2

**`dProbePass(cumDeg)`**（线性插值）：

| 累计探角 | 判通 cm（默认 cfg） |
|----------|---------------------|
| 0° | `DPASS_AT_0` = **26** |
| 45°（3×15°） | `DPASS_AT_MAX` = **40** |
| 对比 | `dProbeOk` ≈ **44**（仅 FORWARD 分段仍用） |

**探通后（`cum > 0`）**：

```
decide pass → ST_WEAVE（SPEED_WEAVE × msWeaveForCumDeg(探角)）
           → ST_RECENTER（反向转回 weaveRecenterDeg）
           → ST_FORWARD（航向与进 DECIDE 前一致）
```

正前直通（`cum=0`）仍直接 `FORWARD`，不 WEAVE。

**串口**：`pass thr=`、`ST -> WEAVE`（含 `cum=`、`ms`）、`RECENTER`、`dPass 0/max=` 上电打印。

### cfg-1.1（加长 WEAVE）

| 项 | cfg-1.0 | cfg-1.1（**默认**） |
|----|---------|---------------------|
| WEAVE | 固定 450ms | **550～950ms**：`320 + 累计探角×14ms` |
| 实测 | 单障 ×5 全过 | 左探 WEAVE 后 **RECENTER 右侧仍蹭柱** → 加长带角直行 |

仍蹭：加大 `MS_WEAVE_PER_DEG` 或 `MS_WEAVE_MIN`（`params_alg_1_3_cfg_1_1.h`）。

**不宜采用（讨论否决）**：左探通过后再**固定 +5°～10°**。探头只证明「当前朝向」前方够宽；若左侧也是窄通道，再多转只会**加大左侧撞障概率**，右侧间隙未必改善。单侧加角 = 假设「左壁无穷远」，与窄缝/双障场景矛盾。

| 场景 | 固定加左角 |
|------|------------|
| 单柱在正前、左侧空（跑道） | 可能略减右缘刮蹭，**无侧向反馈时不可靠** |
| 左壁也近（窄通道、双障） | **左侧更易撞**，弊大于利 |

更合理方向：有侧向测距再决定加角；或 **`dProbePass`** 在「已通过那一档转角」上略提高 cm 门槛（促选更小通过角），而非通过后盲加角。

## ALG-1.1（`fsm_rhr`）— 基线摘要

（状态机、距离分段、`dStop`/`dBrake`/`dSafe`/`dProbeOk` 推导见 [VERSION.md](../hardware/uno_tb6612_hc04/VERSION.md)。）

| 状态 | 行为 |
|------|------|
| `FORWARD` | 分段调速；近 → DECIDE；极近 → BACKUP |
| `DECIDE` | 右 35° / 回正 / 左 35° / ESCAPE |

| `ESCAPE` | 升档 + 大角转 → DECIDE |

FORWARD↔BACKUP 振荡已在 1.1 用冷却 + `NEAR_CONFIRM` / `CLEAR_CONFIRM` 缓解；1.2 在此基础上改探测与后退时长。
