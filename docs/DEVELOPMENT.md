# 避障版开发规划

> 当前固件：`car_tb6612_avoid/car_tb6612_avoid_v2.ino`  
> 上一版基线：`car_tb6612_avoid_v1.ino`（保留，可随时刷回）  
> 硬件基线见根目录 [README.md](../README.md)

## 方向调整说明

最初规划是**舵机扫描 + 扇区测距**，但实际买到的是 **360° 连续旋转舵机**（无绝对角度反馈，无法稳定停在 0°/45°/90°），方案改为：**继续用固定朝前的 HC-SR04 + 软件策略**深耕，再视情况补硬件。

## v2 设计

### 1. 小车机械参数显式建模

避障阈值不再凭感觉填 18/25/32，而是从**车身尺寸 + 制动余量**推导：

```cpp
CAR_LENGTH_CM      = 18   // 车头到车尾
CAR_WIDTH_CM       = 14
CAR_HEIGHT_CM      = 9
SENSOR_OFFSET_CM   = 3    // 探头突出车头多少

BRAKE_PAD_CM       = 12   // 减速余量
STOP_PAD_CM        = 6    // 停车余量
BACKUP_PAD_CM      = 4    // 极近后退余量

DEG_PER_MS_AT_SPEED_LOW = 0.30   // 转向标定：低速每毫秒约多少度
MAX_TURN_DEG = 180
```

派生阈值（运行时打印一次便于核对）：

| 名称 | 公式 | 含义 |
|------|------|------|
| `dStop`  | `STOP_PAD + BACKUP_PAD` | 极近：直接后退 |
| `dBrake` | `CAR_WIDTH + STOP_PAD` | 近障：进入 DECIDE |
| `dSafe`  | `CAR_LENGTH + BRAKE_PAD` | 中距：低速通过 |
| `dClear` | `dSafe + 10` | 路通：可加速 / 复位 stuck |
| `dProbeOk` | `dSafe + 5` | 转向探测判定能走 |

转向时间从「角度」推导，不再写死 `MS_TURN_*`：

```cpp
turnMsForDegrees(deg) = deg / DEG_PER_MS_AT_SPEED_LOW
```

> **标定方法**：架空 / 平地放车，下发 `doTurnInPlace(+1, 90)` 看实际转角；按比例修正 `DEG_PER_MS_AT_SPEED_LOW`。

---

### 2. 距离分段决策（不再「一近就停 + 后退」）

| 距离区间 | 动作 |
|----------|------|
| `cm < dStop` | 直接 **BACKUP** 后退一段，再 DECIDE |
| `dStop ≤ cm < dBrake` | 进入 **DECIDE** 做右手法则探测（**不一定后退**） |
| `dBrake ≤ cm < dSafe` | 低速直行 |
| `dSafe ≤ cm < dClear` | 中速直行 |
| `cm ≥ dClear` | 高速直行；满足时长后复位 `stuckLevel` |

> 与 v1 区别：v1 触发避障一律「停→退→转」；v2 只有**极近**才后退，**近但未抵车头**只转向探测，加快通过性。

---

### 3. 状态机

```
                ┌──────────┐
        上电 →  │   IDLE   │
                └────┬─────┘
                     ▼
              ┌──────────┐
        ┌────│ FORWARD  │◄──────────┐
        │    └────┬─────┘            │
        │ 近障    │                   │
        ▼        │ 路通                │
┌──────────┐     │                    │
│  DECIDE  │     │                    │
│ (右手法则)│     │                    │
└────┬─────┘     │                    │
     │   ┌───────┴──────┐             │
     │  方向OK         │              │
     │   ▼              ▼             │
     │ FORWARD       三面皆堵         │
     │                  │              │
     │           ┌──────▼─────┐       │
     │           │   ESCAPE   │       │
     │           │（升档/反向）│       │
     │           └──────┬─────┘       │
     │ 极近              │             │
     │                  ▼              │
     └─→ ┌──────────┐                 │
         │  BACKUP  │─────────────────┘
         └──────────┘
```

| 状态 | 触发 | 行为 |
|------|------|------|
| `FORWARD` | 默认 | 按距离分段调速；近障→ DECIDE；极近→ BACKUP |
| `DECIDE`  | FORWARD 近障 | 右手法则四步探测（见下） |
| `BACKUP`  | 极近 / DECIDE 中遇极近 | 后退固定时间，再回 DECIDE |
| `ESCAPE`  | DECIDE 三面皆堵 | 升 stuckLevel，反向大角度转，回 DECIDE |
| `IDLE`    | 上电 | 短暂稳定，进入 FORWARD |
| `TURN_PROBE` | 保留 | v2 暂以同步 `doTurnInPlace` 实现，未来改非阻塞 |

---

### 4. 右手法则探测（DECIDE 内部分相位）

不一次到位转一大圈，而是 **小步探测 + 测距确认**：

| 相位 | 动作 | 测距结果 |
|------|------|----------|
| 0 | 进入 DECIDE，先测前方 | 已通 → FORWARD |
| 1 | 右探约 35° | 通 → 保持朝向 → FORWARD（**优先右**） |
| 2 | 回正（撤销右探角度）再测 | 通 → FORWARD |
| 3 | 左探约 35° | 通 → FORWARD |
| 4 | 三面皆堵 → 回正 → ESCAPE | — |

判通阈值：`cm ≥ dProbeOk`（默认 `dSafe + 5`）。

> 与 v1 区别：v1 每次脱困固定「反向转一个角度」，方向选择只看交替；v2 **每次决策都先尝试右**，符合右手贴墙习惯，便于以后走迷宫。

---

### 5. 已知保留 / 后续 (v3+)

- 转向仍依赖**时间法**（受电压、地面影响）→ v4 改编码器或 MPU 偏航角积分。
- DECIDE 是**同步阻塞**实现，转向时无法测距、无法响应中断 → 后续做非阻塞状态机（TURN_PROBE 拆出独立子状态）。
- 暂未维护转向历史 → v3 记录最近 N 次 DECIDE 结果，连续右转失败时跳到 ESCAPE。
- 路径规划仍是反应式，不是真“迷宫求解”。

## 切换 v1 / v2

Arduino 同一 sketch 文件夹内多个 `.ino` 会**一起编译**，所以本目录的两个固件**不能同时启用**。约定如下：

- 编辑 v2：把 `car_tb6612_avoid_v1.ino` **临时重命名**为 `car_tb6612_avoid_v1.ino.bak`（或剪切到其它目录）再编译。
- 回到 v1：反向操作。
- 长期方案：v2 稳定后，删除 / 归档 v1，或开两个 sketch 文件夹（如 `car_tb6612_avoid_v1/`）。
